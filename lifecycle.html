Lifecycle Methods in React.js

Lifecycle methods are special methods provided by React that allow you to perform certain actions at specific points in the life cycle of a component. They enable you to control component initialization, rendering, updating, and destruction.

Why Use Lifecycle Methods?

Lifecycle methods allow you to hook into the different stages of a component's life cycle and execute code accordingly. They provide opportunities to perform actions like initializing state, making API calls, updating the UI, or cleaning up resources.

Using Lifecycle Methods in React

React provides several lifecycle methods that can be used in class components:

  • componentDidMount: Invoked immediately after a component is mounted to the DOM.
  • componentDidUpdate: Invoked immediately after updating occurs. This method is not called for the initial render.
  • componentWillUnmount: Invoked immediately before a component is unmounted and destroyed.
  • render: Invoked to render the component's UI representation.

Example: Lifecycle Methods

Here's an example that demonstrates the usage of some lifecycle methods:


import React from 'react';

class ExampleComponent extends React.Component {
  constructor(props) {
    super(props);
    this.state = {
      count: 0
    };
  }

  componentDidMount() {
    console.log('Component mounted');
  }

  componentDidUpdate() {
    console.log('Component updated');
  }

  componentWillUnmount() {
    console.log('Component will unmount');
  }

  incrementCount() {
    this.setState((prevState) => ({
      count: prevState.count + 1
    }));
  }

  render() {
    return (
      <div>
        <p>Count: {this.state.count}</p>
        <button onClick={() => this.incrementCount()}>Increment</button>
      </div>
    );
  }
}

export default ExampleComponent;

In the above example, the `ExampleComponent` class component defines the lifecycle methods `componentDidMount`, `componentDidUpdate`, and `componentWillUnmount`. The `componentDidMount` method is invoked immediately after the component is mounted to the DOM, `componentDidUpdate` is invoked after updates occur, and `componentWillUnmount` is invoked before the component is unmounted.

The component also initializes state with a `count` property and provides a method to increment the count value when the button is clicked.

Conclusion

Lifecycle methods in React provide powerful hooks to control and manage the life cycle of components. They enable you to perform various actions at specific points in a component's life cycle, allowing for initialization, updating, and cleanup operations.
